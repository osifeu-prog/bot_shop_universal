# מערכת רשת חברתית מוטיבציונית – SLHNET / Buy My Shop

מסמך זה מתאר את שכבת ה-"Social / Motivation" של האקו־סיסטם, שתשב מעל הבוט וה-API הקיימים ותהפוך את הפרויקט הזה ל"פרויקט בית" לכל שאר המערכות.

## 1. תשתית טכנית

- **Backend**: FastAPI (כבר קיים ב-`main.py`) + הרחבות עתידיות
- **Database**: PostgreSQL דרך `db.py` – טבלאות:
  - `users`, `payments`, `referrals`, `rewards` (כבר קיימות)
  - בעתיד: `social_posts`, `social_likes`, `expert_votes` וכו'
- **Frontend**:
  - GitHub Pages דרך `docs/` (האתר ב-`https://slh-nft.com/`)
  - בהמשך: דשבורד דינאמי שצורך נתונים מ-`social_api.py` ו-`slh_public_api.py`
- **Infrastructure**:
  - Railway – להרצת הבוט + ה-API
  - GitHub – ניהול קוד, גיבויים, CI/CD

## 2. מודל משתמשים

כל משתמש בוט/אתר מיוצג ב-DB בטבלת `users`:

- `id` – user_id של טלגרם
- `username`
- `first_seen_at`

בעתיד נוסיף שדות ו/או טבלה מקבילה:

- פרופיל מורחב (שם מלא, אימייל, טלפון)
- מספר נקודות מצטברות
- רמת מומחיות (Level)

## 3. נקודות, דירוג ומוטיבציה

המטרה: לעודד שיתוף, פעילות ותנועת משתמשים:

- הצטרפות חבר חדש דרך לינק ריפרל → נקודות למזמין
- פעולות בתוך המערכת (פוסטים, תגובות, תרומה לידע) → נקודות נוספות
- טבלת דירוג (Leaderboard) – על בסיס:
  - כמות נקודות
  - כמות הפניות (referrals)
  - פעילות חברתית (פוסטים, לייקים)

### סוגי נקודות (דוגמאות):

- +1 נקודה לכל הפניה פשוטה
- +5 נקודות לפעילות איכותית (פוסט שמקבל X לייקים)
- בונוסים תקופתיים על משימות (Quests)

## 4. מנגנון בחירת מומחים

מטרת המערכת: להעלות למעלה את מי שתורם הכי הרבה ערך.

קריטריונים אפשריים למומחה:

- מספר נקודות גבוהה
- תרומה איכותית (פוסטים / תכנים עם הרבה לייקים/תגובות)
- היסטוריית פעילות יציבה

תהליך בחירה:

- אלגוריתם אוטומטי שמדרג לפי נקודות/תרומה
- אפשרות ל"הצבעה קהילתית" (טבלת votes)

תפקיד מומחה:

- להוביל קבוצות דיון
- להפעיל חדרי זום/לייב
- לאשר תוכן של אחרים (Moderation)
- לקבל יותר זכאות לתגמולים (SLH, חשיפה, תנועה לחנות שלו)

## 5. פיצ'רים חברתיים עתידיים

- **Feed חברתי**:
  - פוסטים קצרים בעלי CTA (לינק לחנות, לקורס, לשירות)
  - תגובות ולייקים
- **הודעות פרטיות** (direct messaging)
- **קבוצות וקהילות משנה** (לפי תחום עסקי)
- **אנליטיקה**:
  - מעקב אחרי הזמנות, נקודות, פעילות
  - פרופיל "ערך רשת" לכל משתמש

## 6. API פתוח

על גבי הבסיס הקיים (`slh_public_api.py`, `social_api.py`) נבנה:

- REST API סטנדרטי ל:
  - פרופיל משתמש
  - נקודות ו-Leaderboard
  - פעילות חברתית
- בעתיד: SDK ל-JavaScript/TypeScript לצריכת הנתונים מהאתר / אפליקציות חיצוניות.

## 7. שלבי פיתוח (MVP → הרחבה)

### שלב 1 – MVP (מבוסס על הקיים)

- /start בוט – שער כניסה, רישום משתמש, רפרל
- DB:
  - `users`, `payments`, `referrals`
- /health, /meta – ניטור
- דף נחיתה ב-`docs/index.html`

### שלב 2 – שכבת Social בסיסית

- יצירת טבלאות social_* ב-DB
- הרחבת `social_api.py` עם:
  - `/api/social/profile`
  - `/api/social/referrals`
  - `/api/social/leaderboard`
- הוספת חלק דינאמי ב-`docs/index.html` שמתממשק ל-API

### שלב 3 – מומחים וגיימיפיקציה מתקדמת

- הרחבת מודל המומחים (Experts)
- הוספת Badges, Levels ו-Quests
- חיבור ללוגיקת Rewards (טבלת `rewards`) לצורך SLH / NFT / הטבות אחרות
