# SLHNET – פרויקט הבית (Home Platform)

מטרה: להפוך את מאגר הקוד הזה למערכת בסיס שמרכזת:

- את בוט ה-Gateway (`Buy My Shop`)
- את ה-API הציבורי והפנימי
- את לוגיקת הניקוד, הרפרלים וה-Experts העתידית
- את דפי הנחיתה, המצגות והקונספט של כל האקו־סיסטם

## שכבות בפרויקט

1. **Telegram Gateway Bot** (`main.py`)
   - webhook דרך `/webhook`
   - /start, /investor, /whoami וכו'
   - לוגיקת רפרל, לוגים לאדמין, מונים ותמונת שער

2. **Core APIs**
   - `slh_public_api.py` – API ציבורי לקריאה רק:
     - מצב מערכת
     - נתוני high-level
   - `slh_core_api.py` – API פנימי/מתקדם:
     - ניהול טבלאות
     - הפניות
     - ניקוד בסיסי
   - `social_api.py` – SPI לשכבת Social (בהתאם ל-spec המסביר)

3. **Database Layer** – `db.py`
   - חיבור PostgreSQL
   - יצירת טבלאות:
     - `payments`, `users`, `referrals`, `rewards`, ועוד
   - פונקציות עזר שמסתירות את השאילתות

4. **Docs & Web**
   - `docs/index.html` – דף נחיתה ראשי
   - `docs/specs/` – מסמכי אפיון:
     - `social-logic.md`
     - `social-motivation-network.md`
     - מסמכים נוספים בעתיד
   - `docs/CNAME` – מחבר את GitHub Pages ל-`https://slh-nft.com/`

## אינטגרציה עם שאר פרויקטים

- כל בוט/שירות עתידי של SLHNET יכול:
  - להשתמש באותו בסיס נתונים (או מסד משותף / שארד)
  - להתחבר ל-API הקיים (או להתרחבות שלו)
  - להפנות משתמשים דרך אותו Gateway ראשי

## עקרונות תכנון

- **מודולריות**: כל שכבה יושבת בקובץ/מודול משלה, עם הרבה מקום להתרחבות.
- **אחידות**: אותו סט env vars, אותו מנגנון לוגים, אותה חוויית /start.
- **שקיפות**: חיבור בין:
  - בוט ← DB ← אתר ← APIs ← אקדמיה / טוקן / NFT
- **הרחבה עתידית**:
  - קל להוסיף Routers נוספים
  - קל לחבר Frontend חדש (React / Next.js) מעל ה-API
